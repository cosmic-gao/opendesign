# Tunnel 跨框架路由适配器方案

**版本**: v1.0.0
**状态**: 进行中

Tunnel 是一套基于 Proxy/Wrapper 模式的跨框架服务端路由管理方案，旨在解决 Node.js 框架中路由热更新（HMR）和动态卸载的痛点。

## 核心能力

- **极速热更新**: 通过代理模式实现 O(1) 复杂度的路由替换，无须重启服务或重载底层框架。
- **跨框架支持**: 抽象出统一的 `Adapter` 接口，轻松适配 Express, Koa, Fastify, Hono 等主流框架。
- **动态卸载路由**: 绕开框架本身不支持 `unregister` 的限制，支持将请求穿透 (Fallthrough) 到下层 404 处理链路。
- **极致性能**: 采用字符串组合 Key (`[Method]:[Path]`) 进行 O(1) 的 Map 寻址，0 哈希开销及碰撞风险。
- **异常捕获隔离**: 安全捕获动态路由引发的异常并优雅抛给框架，防止进程崩溃。
