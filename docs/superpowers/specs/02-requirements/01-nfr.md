# 非功能性需求

---

## 1. 性能需求

### 1.1 延迟需求

| 场景 | 指标 | 说明 |
|------|------|------|
| 消息路由 | P99 < 10ms | 单跳消息传递延迟 |
| 节点处理 | P99 < 50ms | 单个节点处理时间 |
| 超步骤执行 | P99 < 500ms | 单个超步骤执行时间 |
| Checkpoint | < 1s | 全量 Checkpoint 耗时 |

### 1.2 吞吐量需求

| 场景 | 指标 | 说明 |
|------|------|------|
| 消息处理 | > 10,000 msg/s | 单节点消息处理能力 |
| 并发连接 | > 1,000 | 单进程最大并发连接 |
| 图规模 | 10 - 10,000 节点 | 支持的图规模 |

### 1.3 资源需求

| 资源 | 指标 |
|------|------|
| CPU | 每节点 1-2 核 |
| 内存 | 每节点 256MB - 2GB |
| 网络 | > 100 Mbps |

---

## 2. 可用性需求

### 2.1 可用性指标

| 指标 | 目标 |
|------|------|
| Uptime | 99.9% |
| 故障恢复时间 | < 30s |
| Checkpoint 恢复时间 | < 60s |

### 2.2 容错能力

| 能力 | 说明 |
|------|------|
| 节点故障 | 单节点故障不影响整体 |
| 进程故障 | 可从 Checkpoint 恢复 |
| 网络分区 | 消息本地缓冲，恢复后重发 |
| 数据丢失 | At-Least-Once 语义 |

---

## 3. 扩展性需求

### 3.1 水平扩展

| 维度 | 指标 |
|------|------|
| 节点扩展 | 支持横向扩展 |
| 图规模 | 10 - 10,000 节点 |
| 用户/线程 | 支持多用户/多线程隔离 |

### 3.2 垂直扩展

| 维度 | 指标 |
|------|------|
| 端点数量 | 每节点支持多个端点 |
| 消息大小 | 单消息最大 1MB |
| 缓冲区大小 | 每端点缓冲区可配置 |
