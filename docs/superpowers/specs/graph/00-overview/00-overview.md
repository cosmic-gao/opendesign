# 执行摘要

---

## 1. 项目概述

**通用图计算框架** - 层次化、分布式、可扩展的图计算运行时，支持代码关系分析、逻辑编排、AI Agent 工作流等多种场景。

---

## 2. 核心特性

| 特性 | 设计决策 |
|------|---------|
| **执行模型** | BSP (Bulk Synchronous Parallel) 超步骤模型 |
| **状态管理** | 节点分片 + 消息传递的混合模式 |
| **容错机制** | 增量 Checkpoint + 超步骤边界恢复 |
| **端点设计** | 多 Endpoint 同方向 + 类型安全 |
| **技术栈** | TypeScript/Node.js + Redis + PostgreSQL + Kafka |
| **部署模式** | 多进程/多机容器化部署 (Docker + Kubernetes) |

---

## 3. 设计目标

1. **通用性**: 一个框架支持多种场景（代码关系、逻辑编排、AI工作流）
2. **可扩展性**: 支持横向扩展，适应不同规模
3. **容错性**: 支持断电/故障恢复，Long-running 任务不中断
4. **可观测性**: 提供完整的监控、日志、调试能力
5. **易用性**: 简洁的 API，降低使用门槛

## 5. 关键架构特点

### 5.1 多 Endpoint 同方向设计

```
节点可以拥有多个同方向的端点：

输入端点: in:user, in:tool, in:memory, in:control
输出端点: out:llm, out:tool, out:memory, out:error
```

### 5.2 混合状态管理

- **子图内**：集中式 State（简化编程）
- **跨子图**：消息传递（解耦）

### 5.3 增量 Checkpoint

- 每个 superstep 边界保存状态 diff
- 每 N 个 superstep 保存一次全量 base
- 支持 replay 恢复
