# 目标场景

---

## 1. 场景概述

本框架旨在支持三种核心场景：

```
┌─────────────────────────────────────────────────────────────────┐
│                      通用图计算框架                               │
├───────────────┬───────────────┬─────────────────┬───────────────┤
│   代码关系     │   逻辑编排     │    AI 工作流     │    (扩展)     │
│   (Code)      │   (Logic)     │    (AI Agent)   │              │
├───────────────┼───────────────┼─────────────────┼───────────────┤
│ • AST 依赖    │ • 条件分支    │ • LLM 调用      │ • 实时数据流  │
│ • 模块关系    │ • 循环迭代    │ • 工具调用      │ • 事件驱动    │
│ • 引用追踪    │ • 并行执行    │ • 记忆管理      │ • ...        │
│ • 影响分析    │ • 状态机      │ • 多 Agent 协作 │              │
└───────────────┴───────────────┴─────────────────┴───────────────┘
```

---

## 2. 场景 1: 代码关系图 (Code Graph)

### 2.1 用途

分析代码库的结构、依赖关系、影响范围

### 2.2 节点类型

| 类型 | 说明 | 示例 |
|------|------|------|
| `code:file` | 源代码文件 | `User.ts`, `index.js` |
| `code:function` | 函数/方法 | `function calculate()` |
| `code:class` | 类 | `class UserService` |
| `code:interface` | 接口 | `interface IRepository` |
| `code:module` | 模块 | ES Module, CommonJS |

### 2.3 关系类型

| 关系 | 说明 |
|------|------|
| `import` | 导入关系 |
| `call` | 函数调用 |
| `inherit` | 继承关系 |
| `implement` | 实现关系 |
| `reference` | 引用关系 |

### 2.4 典型操作

- **依赖分析**: 分析模块间的依赖关系
- **影响分析**: 变更某个文件后，影响哪些文件
- **循环依赖检测**: 检测代码中的循环依赖
- **重构模拟**: 模拟重构后的影响范围

### 2.5 规模

- 小型项目: < 1000 文件
- 中型项目: 1000 - 10000 文件
- 大型项目: > 10000 文件

## 3. 场景 2: 逻辑编排图 (Logic Graph)

### 3.1 用途

业务流程编排、条件分支、状态机

### 3.2 节点类型

| 类型 | 说明 |
|------|------|
| `logic:condition` | 条件分支（if/switch/match） |
| `logic:loop` | 循环（while/for/foreach） |
| `logic:parallel` | 并行执行 |
| `logic:sequence` | 顺序执行 |
| `logic:try-catch` | 错误处理 |
| `logic:compensation` | 补偿事务 |

### 3.3 关系类型

| 关系 | 说明 |
|------|------|
| `control` | 控制流 |
| `data` | 数据流 |
| `condition` | 条件分支 |

### 3.4 典型操作

- **条件路由**: 根据条件选择分支
- **并行执行**: 多个任务并行处理
- **事务管理**: Saga/2PC 模式
- **熔断降级**: 故障时降级处理

### 3.5 规模

- 简单流程: < 10 节点
- 中等流程: 10 - 100 节点
- 复杂流程: > 100 节点

---

## 4. 场景 3: AI 工作流图 (AI Agent Graph)

### 4.1 用途

构建 AI Agent、多步骤推理、工具调用

### 4.2 节点类型

| 类型 | 说明 |
|------|------|
| `ai:llm` | LLM 调用节点 |
| `ai:tool` | 工具执行节点 |
| `ai:memory` | 记忆管理节点 |
| `ai:router` | 路由决策节点 |
| `ai:human` | 人工介入节点 |

### 4.3 关系类型

| 关系 | 说明 |
|------|------|
| `agent:request` | Agent 间请求 |
| `agent:response` | Agent 间响应 |
| `agent:delegate` | 任务委托 |
| `tool:call` | 工具调用 |
| `memory:store` | 记忆存储 |
| `memory:retrieve` | 记忆检索 |

### 4.4 典型操作

- **推理循环**: ReAct、CoT 等推理模式
- **工具选择**: 根据上下文选择工具
- **记忆检索**: RAG、长期记忆
- **人机协作**: Human-in-the-loop

### 4.5 规模

- 单 Agent: 1 个 LLM 节点 + 若干工具节点
- 多 Agent: 2 - 10 个 Agent 协作
- 复杂系统: > 10 个 Agent

---

## 5. 场景共同需求

### 5.1 状态管理

- 节点需要维护自己的状态
- 状态需要在超步骤边界同步
- 支持 Checkpoint 和恢复

### 5.2 消息传递

- 节点间需要传递消息
- 消息需要路由到正确的端点
- 支持多种消息类型

### 5.3 容错处理

- 支持节点故障检测
- 支持从 Checkpoint 恢复
- 支持超时处理
